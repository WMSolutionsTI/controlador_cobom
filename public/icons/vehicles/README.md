# Vehicle Icons Structure

This directory contains dynamic vehicle icons organized by modality and status.

## Directory Structure

```
vehicles/
├── incendio/
│   ├── disponivel.png
│   ├── qti.png
│   ├── local.png
│   ├── qti-ps.png
│   ├── regresso.png
│   ├── baixado.png
│   └── reserva.png
├── resgate/
│   └── ...
├── salvamento/
│   └── ...
└── abastecimento/
    └── ...
```

## File Naming Convention

All icon files must follow the pattern:
```
{status-in-lowercase-with-hyphen}.png
```

### Status Examples:
- `disponivel.png` - Vehicle available
- `qti.png` - Vehicle responding to incident
- `local.png` - Vehicle at location
- `qti-ps.png` - Vehicle responding with medical support
- `regresso.png` - Vehicle returning to base
- `baixado.png` - Vehicle out of service
- `reserva.png` - Vehicle in reserve

## Image Specifications

- **Format**: PNG with transparent background
- **Size**: 112px (width) × 80px (height) or similar proportions
- **Orientation Guidelines**:
  - Available: Facing right (→)
  - Returning: Facing left (←)
  - QTI: Front or 3/4 angle
  - Local: Side view or parked
- **Style**: Minimalist, vector-based, easily identifiable

## Usage

The application will automatically load icons based on the vehicle's modality and status:

```
/icons/vehicles/{modality-slug}/{status-slug}.png
```

If a specific icon doesn't exist, the system falls back to the default modality icon stored in the database.

## Modality Slug Mapping

Common modalities and their slugs:
- "Incêndio" → `incendio`
- "Resgate" → `resgate`
- "Salvamento" → `salvamento`
- "Abastecimento" → `abastecimento`
- "Auto Bomba Tanque" → `auto-bomba-tanque`
- "Unidade de Resgate" → `unidade-de-resgate`

The slug is automatically generated by:
1. Converting to lowercase
2. Removing accents
3. Replacing spaces with hyphens
